# Kasa Project

Kasa is a peer-to-peer rental app allowing users to browse various ads. 

## Introduction

This project is built using React, React Router, and Vite. It includes several components designed to be reusable and type-safe using PropTypes.

## Project Structure

The project structure follows a well-organized layout for ease of navigation:

project-root/
|-- src/
|-- |-- assets/      // Static assets such as images, icons, etc.
|   |-- components/  // Reusable UI components and their related styles
|   |-- datas/       // mocked data file
|   |-- pages/       // Page-level components
|   |-- styles/      // global style 
|   |-- main.jsx     // Main entry point of the application
|-- public/          // Static assets such as images, icons, etc.
|-- index.html       // HTML template file
|-- package.json     // Project dependencies and scripts
|-- vite.config.js   // Vite configuration file

## Prerequisites

Before starting with this project, make sure you have the following installed on your machine:

- Node.js (version 14.x or higher)
- npm (version 6.x or higher) or yarn
## Installation and Setup

To install and set up the project locally:

### Clone the repository:
git clone https://github.com/amandinedev/Kasa-app.git
cd Kasa-app

### Install dependencies:
Using npm:
npm install

Or using yarn:
yarn install

### Start the development server:
Using npm:
npm run dev

Or using yarn:
yarn dev

## Usage
After starting the development server, you can navigate to `http://localhost:3000` in your web browser. The application should now be running and accessible.

## Components
The `@components` directory contains reusable UI components that can be used throughout the project.

## Pages
The `@pages` directory holds different pages of the application, each corresponding to a route in the React Router setup.

## Datas
The `@datas` directory is used for managing data, here it's mocked data for development.
